# Value Interrogation

Value Interrogation is a Livingtelligence phenomenon and practice.

It appears when AI adoption forces a person, team, profession, or organization to ask what values automation will amplify.

## Core Question

> What values are we scaling?

This question matters because AI does not automate neutral processes.

It can amplify the values, incentives, assumptions, standards, and blind spots already present in a practice.

## What It Describes

Value Interrogation appears when the pressure to automate, delegate, or scale work with AI creates a deeper institutional question:

- What should be preserved?
- What should be redesigned?
- What should not be automated yet?
- What human goods does this practice protect?
- What would be harmed if this process became faster but thinner?

It is not resistance to technology.

It is the threshold at which AI adoption becomes deliberate rather than reactive.

## Related Shadow

When value interrogation is absent, an organization may experience value abdication.

In that vacuum, other forces decide:

- efficiency
- throughput
- technical convenience
- vendor defaults
- dashboard metrics
- competitive pressure
- inherited process assumptions

The organization may still speak the language of values, while its operating system is governed by unexamined defaults.

## Signals

Possible signals include:

- AI adoption is justified mainly by speed, cost, or productivity.
- Leaders cannot clearly name which human goods the implementation should preserve.
- A team asks what can be automated before asking what should be preserved.
- People resist automation but struggle to articulate the value they are protecting.
- Processes are automated without examining embedded assumptions.
- Metrics become the implicit definition of value.

## Mitigation And Nourishment Assets

Potential assets include:

- Value Interrogation Workshop
- Encoded Values Diagnostic
- Dignity-Centered AI Governance Principles
- AI-as-Medium Practice
- stewardship conversations before automation
- redesign-before-automation protocols

## Public Line

The question is not only what can be automated.

It is what values we want automation to amplify.
